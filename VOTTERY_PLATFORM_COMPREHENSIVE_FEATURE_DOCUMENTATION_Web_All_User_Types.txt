General Users are the core participants of the Vottery platform. They register, vote in elections, earn Vottery Points (VP), compete on leaderboards, interact socially, and win prizes.
──────────────────────────────────────────────── 1.1 CORE VOTING FEATURES ────────────────────────────────────────────────
Available on: Web App and Mobile App
1. Plurality Voting Role: Standard single-choice voting. Users select one option from a list of candidates or choices in an election.
2. Ranked Choice Voting Role: Users rank multiple options in order of preference. The platform processes ranked ballots to determine the winner using elimination rounds.
3. Approval Voting Role: Users approve one or more options simultaneously. All approved options receive a vote, enabling nuanced preference expression.
4. Plus/Minus Voting Role: Sentiment-based voting where users assign positive (+) or negative (−) scores to each option. A statistical significance panel displays real-time confidence levels.
5. MCQ Pre-Voting Quiz Role: Users answer a multiple-choice quiz before their vote is cast. Correct answers may influence VP rewards or unlock voting access. Designed to promote informed participation.
6. MCQ Image Voting Role: Visual voting interface where each election option is represented by an image. Users vote by selecting their preferred image option.
7. Prediction Pool Voting Role: Users wager VP on the predicted outcome of an election. The VP pool is distributed among correct predictors after the election closes.
8. Collaborative Voting Room Role: Real-time group voting environment where multiple users vote simultaneously. Includes live discussion threads, participant emoji reactions, and real-time result streaming. Used for group decision-making sessions.
9. Location-Based Voting Role: Geo-targeted elections appear for users based on their detected geographic location. Enables regional, city-level, or neighbourhood-specific elections.
10. External Voter Gate Role: Allows non-registered users to participate in specific elections via a shared link. Captures partial participation data and prompts registration post-vote.
──────────────────────────────────────────────── 1.2 VOTE INTEGRITY & SECURITY ────────────────────────────────────────────────
Available on: Web App and Mobile App
11. Cryptographic Vote Receipts Role: Status: ⚠️ Partial (cryptographic verification exists, but SHA-256 + RSA receipt spec is not fully confirmed end-to-end). Every vote generates a unique SHA-256 + RSA cryptographic receipt. Users can use this receipt to verify their vote was counted correctly without revealing how they voted.
12. Vote Anonymity via Mixnet Role: Status: ❌ Not implemented/verified (mixnets/ZK-style privacy routing not wired in current Web implementation). All votes pass through an onion-routing mixnet system that removes the link between voter identity and vote content. Mixing cycles run on a configurable schedule.
13. Zero-Knowledge Proofs (ZK-SNARKs) Role: Status: ❌ Not implemented/verified (ZK proofs not wired in current Web implementation). Allows the platform to mathematically prove a vote was cast correctly and counted accurately without revealing the vote content. Protects voter privacy at a cryptographic level.
14. Homomorphic Encryption Role: Status: ❌ Not implemented/verified (homomorphic tallying not wired in current Web implementation). Enables vote tallies to be computed directly on encrypted data. No individual vote is ever decrypted during the counting process.
15. Threshold Cryptography Role: Status: ❌ Not implemented/verified (threshold decryption not wired in current Web implementation). Encryption keys are split across multiple parties using Shamir Secret Sharing. No single party can decrypt election results alone, preventing single points of compromise.
16. Blockchain Vote Verification Portal Role: Status: ✅ Fully implemented (public verification/audit portals exist in the Web app). Every vote is recorded on a tamper-evident public ledger. Users can verify their vote's inclusion and integrity via the public verification portal.
17. VVSG Compliance Role: Status: ⚠️ Partial/Not verified (institutional VVSG compliance is not fully evidenced end-to-end in the Web codebase). The platform meets Voluntary Voting System Guidelines — the same standards applied to national election systems — ensuring institutional-grade vote integrity.
18. Vote Verification Portal Role: Status: ✅ Fully implemented (verification portal route and flow exist in the Web app). User-facing portal where any voter can enter their cryptographic receipt to independently verify their vote was recorded and counted correctly.

──────────────────────────────────────────────── 1.4 GAMIFICATION AND VP ECONOMY ────────────────────────────────────────────────
Available on: Web App and Mobile App
27. Vottery Points (VP) Universal Currency Role: VP is the platform's internal economy currency. Earned by voting, completing quests, maintaining streaks, and winning events. Spent on participation fees, marketplace items, and redemptions. The VP Universal Currency Center is the central hub for all VP activity.
28. VP Earning Actions Role: Users earn VP through: casting votes (+5 to +50 VP per vote), daily logins (+10 VP), streak bonuses (multiplier increases per consecutive day), quest completions (+50 to +500 VP), referrals (+100 VP per successful referral), and winning prediction pools (variable).
29. VP Spending and Redemption Role: VP can be spent on: election participation fees, quest packs from the marketplace, cryptocurrency conversion, charity donations, gift card purchases, and VIP tier access unlocks.
30. VP Redemption Marketplace Role: Storefront where users exchange accumulated VP for tangible rewards including quest packs, real-world experiences, gift cards, and charitable contributions.
31. VP Charity Hub Role: Users can donate VP to a curated list of supported charitable organisations directly from their wallet.
32. VP Crypto Conversion Role: Users convert VP to supported cryptocurrency at the current exchange rate. Processed via the platform's crypto exchange panel.
33. XP and Leveling System Role: Users earn Experience Points (XP) alongside VP. XP accumulates to advance through 100 levels. Each level milestone unlocks new features, higher VP multipliers, and exclusive badges.
34. Achievement Badge System Role: 50+ unlockable badges awarded for platform milestones (first vote, 100-day streak, prediction accuracy, community contributions, etc.). Displayed prominently on user profile.
35. Streak Tracking System Role: Tracks consecutive daily voting activity. Streak multipliers increase VP earnings exponentially. Users receive smart notifications 24 hours and 1 hour before a streak expires.
36. Dynamic Quest Management Dashboard Role: Central hub for all active AI-generated quests. Displays progress, objectives, VP rewards, difficulty level, and time remaining per quest.
37. AI-Generated Quests (GPT-4) Role: GPT-4 generates personalized quests based on each user's voting history, level, engagement score, and preferred categories. Quest types include voting streaks, category explorers, prediction challenges, and social quests.
38. Seasonal Challenges Role: Time-limited platform-wide challenge events tied to seasons, world events, or platform milestones. Available in 61 languages with region-specific variations.
39. Prediction Pools System Role: Users wager VP on election outcome predictions. A live leaderboard tracks predictions in real-time. VP pool resolves automatically when the election closes, distributed to correct predictors proportionally.
40. 3D Gamified Election Experience Center Role: Immersive 3D slot machine visualization for high-stakes elections with jackpot display, prize reveal animations, and celebration sequences.
41. Gamified Feed Overlay System Role: In-feed achievement pop-ups, animated VP reward notifications, quest progress bars, and streak trackers embedded directly within the main content feed. Mini-games accessible from the feed.
42. Platform Gamification Widget Role: Persistent widget visible across all screens showing VP balance, active streak count, and nearest quest milestone. Provides constant engagement reinforcement.
43. Unified Gamification Dashboard Role: Single-screen overview of all gamification elements: VP balance, active challenges, leaderboard position, quick action buttons, and recent achievement unlocks.

Available on: Web App and Mobile App
The Vottery notification system spans seven delivery channels, with ML-optimized send timing and multi-channel routing intelligence.
244. Email Notifications (Resend via Supabase) Role: Transactional and marketing email delivery via Resend. Claude 3 Sonnet generates personalized email body content. GPT-4 optimizes subject lines via A/B testing. Use cases: onboarding sequences, prize notifications, payment receipts, compliance documents, subscription billing, and executive reports.
245. SMS Notifications (Telnyx Primary, Twilio Fallback) Role: Time-sensitive SMS delivery for: two-factor authentication codes, streak expiration warnings, winner announcements, critical security alerts, and payment failure notifications. Automatic failover from Telnyx to Twilio if primary provider experiences degradation.
246. Push Notifications (Web Push) Role: Browser and mobile push notifications for real-time events: new elections in preferred categories, social interactions, VP transactions, quest milestones, and streak reminders. ML-timed delivery per user.
247. In-App Notifications (Supabase Realtime) Role: Real-time in-app notification delivery via Supabase Realtime WebSocket connections. Sub-second delivery for: VP transactions, social reactions, comment replies, friend requests, and live election events.
248. WebSocket Live Events Role: Status: ⚠️ Partial (realtime streaming exists, but “sub-100ms everywhere” parity is not fully evidenced across all dashboards). Real-time data streaming for live platform events: live election results updating in real-time, fraud alert streaming to admin dashboards, live collaborative voting room updates, and live leaderboard position changes.
249. Slack and Discord Integration Role: Webhook-based Slack and Discord notifications for: brand advertiser budget threshold alerts, admin team operational alerts, fraud detection notifications, and system performance alerts. Configurable per admin role and alert type.
250. WhatsApp Notifications (Twilio) Role: High-priority WhatsApp message delivery via Twilio for: critical account security alerts, major prize win announcements, and time-sensitive payment notifications for users in markets where WhatsApp is the primary communication channel.

Available on: Web App and Mobile App
273. Email and Password Authentication Role: Standard account creation and login via email address and password. Managed by Supabase Auth with bcrypt password hashing.
274. Google OAuth Role: One-tap sign-in and registration using an existing Google account. No password required.

279. IP Geolocation Security Role: Geographic location verification for all sessions. Flags access from anomalous locations (significant geographic jumps within short time windows) as potential account compromise.
280. SQL Injection Detection Role: Real-time scanning of all incoming requests for SQL injection patterns. Blocks malicious queries before they reach the database layer.
281. CORS Protection Role: Cross-origin resource sharing whitelist enforcement. All API requests from non-whitelisted origins are rejected with appropriate CORS error responses.
282. Webhook Replay Protection Role: Timestamp and cryptographic signature verification on all incoming webhooks. Prevents replay attacks where valid webhook payloads are resent maliciously.
